# Instrument Definition Authoring (v1.8, experimental)

合言葉: **「instrument YAML を作りやすくする」**

v1.7 で **空 directory から pack の雛形** が作れるようになった。
v1.8 では pack の中身、つまり **instrument YAML を category 別 template
から scaffold** できるようにし、pack に **`add-instrument`** で
登録するところまで CLI 化する。

> v1.8 でも MCP tool 追加ゼロ (Stable 43 / Experimental 7 / 合計 50 不変)。
> PDF → YAML 自動抽出 / LLM 自動定義生成 / 実機自動検証 / remote
> registry / Python plugin / backend plugin にはまだ進まない。

## CLI

```bash
# 単体 instrument YAML を生成 (pack 非依存)
visa-mcp instrument scaffold power_supply \
    --output instruments/kikusui_pmx.yaml \
    --manufacturer "Kikusui" --model "PMX-A"

# pack に instrument を追加 (scaffold + extension.yaml 更新 + registry index)
visa-mcp extension add-instrument my_pack \
    --id kikusui_pmx --category power_supply \
    --manufacturer "Kikusui" --model "PMX-A"

# 変更予定だけ確認
visa-mcp extension add-instrument my_pack \
    --id kikusui_pmx --category power_supply --dry-run --json
```

## 役割分担

| CLI | 対象 | 用途 |
|-----|------|------|
| `instrument scaffold` | 単体 YAML 1 件 | pack 外で instrument YAML を量産 / 試作 |
| `extension add-instrument` | pack 内 | scaffold + `extension.yaml.contents.instruments` 追記 + `registry_entries/INDEX.yaml` entry 追加 + 更新後 validate |

両方とも生成 instrument は **`support_level: draft`** 固定。
`metadata.manual_ref` に必ず TODO placeholder が入る。
生成 YAML は `validate instrument` を error 0 で通る。

## Category templates (v1.8)

| Category | 提供される雛形 |
|----------|----------------|
| `power_supply` | identify / reset / set_voltage(+verify) / query_voltage(polling_safe) / set_current_limit(+verify) / query_current_limit(polling_safe) / measure_voltage / measure_current / set_output / query_output + **safety.ratings + preconditions + safe_shutdown + state_query** |
| `dmm` | identify / reset / measure_voltage_dc(polling_safe) / measure_voltage_ac / measure_current_dc / measure_resistance + state_query |
| `temperature_meter` | identify / measure_temperature / measure_temperature_channel(param) + state_query |
| `generic_scpi` | identify / reset / query_errors (skeleton) |

特に `power_supply` template は **AI agent 向けに安全な構造**を最初から
含む:
- `safety.ratings` (rated / absolute_max / recommended_max / unit)
- `safety.preconditions` (`set_output ON` 前に voltage / current_limit
  必須)
- `safe_shutdown` (`OUTP OFF` → `VOLT 0`)
- `state_query` (voltage_set / current_limit / measured / output_state +
  ON/OFF map)
- `verify` (set_voltage / set_current_limit に read-back tolerance 付き)

## draft ポリシー

生成 YAML は冒頭に以下のコメントが付く:

```
# Generated by `visa-mcp instrument scaffold` (v1.8).
#
# THIS IS A DRAFT DEFINITION.
# Do not use with real hardware until the following are reviewed:
#   - commands / SCPI strings (verify against manufacturer manual)
#   - safety limits (ratings / preconditions)
#   - safe_shutdown sequence
#   - verify (read-back) settings
#   - state_query mapping
#   - response_format unit / parsing
```

`scaffold_instrument_definition()` も `add_instrument_to_pack()` も
常に `instrument_definition_draft` warning を返す (CONTRIBUTING.md
ポリシーと整合)。

## `add-instrument` の安全策

| 状況 | 挙動 |
|------|------|
| pack directory が存在しない / `extension.yaml` 不在 | error |
| pack の `extension.yaml` が事前 validate で error | error (修正後 retry) |
| `instrument_id` が小文字英数 + `_` / `-` 以外 | error |
| 既存 instrument file 存在 | `--force` 必須 |
| registry id 重複 | **`--force` でも拒否** |
| 更新後 validate 失敗 | 全 file を rollback |
| `--dry-run` | file 書き込み・lockfile 編集を一切しない |

更新は **3 file**: `instruments/<id>.yaml` 生成 + `extension.yaml` の
`contents.instruments` / `contents.registry_entries` 追記 +
`registry_entries/INDEX.yaml` に entry 追加。

## Authoring workflow (instrument 単位)

```
visa-mcp instrument scaffold power_supply \
    --output instruments/kikusui_pmx.yaml \
    --manufacturer Kikusui --model PMX-A
  ↓ (review manual_ref / safety limits / SCPI vs manual / verify)
visa-mcp validate instrument instruments/kikusui_pmx.yaml
  ↓ (場合により --strict 相当の lint)
# pack に登録するなら
visa-mcp extension add-instrument my_pack \
    --id kikusui_pmx --category power_supply \
    --manufacturer Kikusui --model PMX-A
  ↓
visa-mcp extension doctor my_pack/extension.yaml
visa-mcp extension package my_pack/extension.yaml --dry-run
```

## `manual_ref` の書き方推奨 (v1.8.1 docs 補強)

`metadata.manual_ref` は **str** field。registry 掲載 / PR review で
出典確認の起点になる。最低限以下を含めると良い:

```yaml
metadata:
  # 推奨フォーマット (1 行)
  manual_ref: "<vendor> <model> <doc title>, Rev <X.YZ>, pp.<NN-NN>"
```

具体例:

```yaml
manual_ref: "Kikusui PMX-A Programming Manual, Rev 1.05, pp.34-58"
manual_ref: "Keysight 34460A/34461A Programmer's Guide, Edition 5, MEASure subsystem"
manual_ref: "Rigol DG1000Z Programming Guide, 2019-01, p.12 (BURSt)"
manual_ref: "https://download.tek.com/manual/MDO3000-Programmer-Manual-EN.pdf p.45"
```

URL でも可。CONTRIBUTING.md のポリシー (manual / SCPI 表の出典明示 /
NDA 内容を公開 PR に含めない) と一致。`scaffold` は生成時に
`"TODO: URL or document title + revision + page range"` placeholder を
入れるので、PR 前に必ず実体に置き換えること。

## category 名 (v1.8.1 docs)

CLI `--category` と instrument YAML `metadata.category` は **同じ値**:

| CLI / metadata.category | 説明 |
|-------------------------|------|
| `power_supply` | プログラマブル電源 |
| `dmm` | デジタルマルチメーター |
| `temperature_meter` | 温度計 / データロガー |
| `generic_scpi` | カテゴリ未確定 / 汎用 SCPI |

v1.8.1 で `dmm` template の `metadata.category` を `multimeter` から
`dmm` へ統一 (CLI 名と一致)。既存の `multimeter` 表記の instrument YAML
を持つ pack は引き続き動くが、新規 scaffold は `dmm` で揃える。

## `--force` の安全策 (v1.8.1)

`instrument scaffold --force` で既存 YAML を上書きする場合、誤って
手編集内容を消さないよう **`.bak-<UTC ts>` backup** が自動で作られる。

```
[OK] scaffold power_supply -> instruments/psu.yaml
  WARN  instrument_scaffold_force_backup: --force: existing file backed
        up to psu.yaml.bak-20260524T120000Z (remove manually after review)
```

backup file の cleanup は **手動**。review 後に不要なら削除すること。

`extension add-instrument --force` も同様 (instrument YAML への backup
は scaffold 層で行われる)。なお **registry id 重複は `--force` でも
拒否**するポリシーは v1.8 から不変。

## `support_level` 昇格の目安

| level | 条件 |
|-------|------|
| `draft` | scaffold 直後 / 未検証 |
| `experimental` | manual 参照で SCPI が正しいことを目視確認 |
| `tested` | mock または実機で identify / 主要 query / set が動く |
| `verified` | 実機で identify / safety / verify / safe_shutdown / state_query 全て確認、`metadata.validation_evidence` を埋める |

`verified` 昇格は **LLM 単独で決定しない**こと
([CONTRIBUTING.md](../CONTRIBUTING.md) 参照)。

## v1.8 で対応しない (v1.9+ 候補)

- PDF / datasheet → YAML 自動抽出の本格化
- LLM による完全自動生成 (人間 review 前提)
- 実機自動検証 (CI hardware loop)
- remote registry / pull CLI
- Python plugin / backend plugin
- `instrument lint` 専用 CLI (現状は `validate instrument` で代替)
